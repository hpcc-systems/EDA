MODULE Side-Effect Actions

Actions contained in the MODULE have the following semantics:

1) Any action immediately preceding an attribute is added as a side-effect of that attribute.

2) Actions cannot precede the following structured attribute types: TRANSFORM, RECORD, MODULE, TYPE, and MACRO.

3) Generally the action will be executed once if the attribute is used. Where attributes are used in a conditional context (for example, inside PERSISTed attributes or conditional actions) they may be executed more than once.
